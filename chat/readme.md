# Fuel Monitoring Microservice

An AI-powered microservice for monitoring fuel station stock, alerts, and trends using Gemini + FastAPI.

---

# Features

* AI chat interface powered by Gemini
* Tool-calling / agentic workflow
* Fuel stock monitoring
* Alert management
* Consumption trend analysis
* Interactive Swagger API documentation
* Mock data support for local development

---

# Project Setup

## 1. Create a Python Virtual Environment

```bash
python3 -m venv venv
source venv/bin/activate
```

After activation, your terminal prompt should start with:

```bash
(venv)
```

---

## 2. Install Dependencies

```bash
pip install -r requirements.txt
```

The first installation may take 1–2 minutes.

---

## 3. Configure Gemini API Key

Copy the environment template:

```bash
cp .env.example .env
```

Open the file:

```bash
nano .env
```

Replace:

```env
GEMINI_API_KEY=your_gemini_api_key_here
```

with your real Gemini API key.

Get a free API key from:

[Google AI Studio API Keys](https://aistudio.google.com/app/apikey?utm_source=chatgpt.com)

Save and exit Nano:

* `Ctrl + O` → Enter
* `Ctrl + X`

---

# Running the Service

Start the microservice:

python main.py

You should see:

# Using the `/chat` Endpoint

## Send Your First Request

### Step 1 — Open `POST /chat`

Expand the endpoint section.

### Step 2 — Click `Try it out`

Located at the top-right of the endpoint panel.

### Step 3 — Replace the Example Body

Use:

```json
{
  "message": "What is the current fuel stock at all stations?",
  "history": [],
  "session_id": "demo-1"
}
```

### Step 4 — Click `Execute`

### Step 5 — View the Response

Scroll down to see the generated Gemini response.

# Multi-turn Conversation Example

After asking about alerts, continue the conversation using chat history:

```json
{
  "message": "What should I do about the low Essence stock?",
  "history": [
    {
      "role": "user",
      "content": "Are there any critical alerts I should know about?"
    },
    {
      "role": "assistant",
      "content": "PASTE THE RESPONSE YOU GOT HERE"
    }
  ],
  "session_id": "demo-1"
}
```

This demonstrates conversational memory and context handling.

# Understanding the Agentic Workflow

When Gemini decides to use a tool, terminal logs will display messages like:

```text
INFO | Tool call: get_current_stock({'station_id': 'AGIL-001'})
INFO | Tool call: get_alerts({})
```

The workflow is:

1. Gemini analyzes the user request
2. The agent selects the appropriate tools
3. The service retrieves mock or real data
4. Gemini synthesizes the final response

This is the complete agentic loop in action.
